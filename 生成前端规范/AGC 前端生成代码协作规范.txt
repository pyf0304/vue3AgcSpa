【文档标题】
AGC 前端生成代码协作规范（GeneCode 模块）
【文档元信息】
1.	版本：v1.0
2.	生效日期：2026-04-22
3.	适用范围：GeneCode 模块（后续可推广到全项目）
4.	维护人：前端负责人 + 代码生成器维护人
5.	评审周期：每月一次
【1. 目标与原则】
1.1 目标
1.	保证代码可重生成，不丢业务能力。
2.	降低自动生成代码与人工改动冲突。
3.	提升缓存一致性与联动可靠性。
1.2 核心原则
1.	生成层可重建。
2.	扩展层承载业务差异。
3.	跨表联动统一入口。
4.	规则先行，变更可追踪。
【2. 分层定义】
2.1 生成层（不可手改业务）
1.	实体定义。
2.	WApi 基础调用。
3.	Base CRUD 流程。
2.2 扩展层（可手改业务）
1.	Ex 类中的业务规则。
2.	缓存联动策略。
3.	页面交互策略。
2.3 配置层（可演进）
1.	默认值。
2.	筛选开关。
3.	排序与展示策略。
【3. 可变化与不可变化判定】
3.1 不可变化项
1.	字段契约。
2.	接口参数与返回结构。
3.	主键与唯一性框架。
4.	生成器约定目录结构。
3.2 可变化项
1.	默认查询条件。
2.	是否显示、是否在用等业务筛选。
3.	弹窗流程、同页编辑流程。
4.	缓存刷新联动规则。
3.3 判定示例
1.	需求：CodeType 改动后刷新 vCodeType_Sim。
2.	判定：属于业务联动策略。
3.	放置层：扩展层。
4.	结论：不应放生成 WApi 文件。
【4. 缓存联动规范】
4.1 统一规则
1.	增加成功后必须联动刷新。
2.	修改成功后必须联动刷新。
3.	删除成功后必须联动刷新。
4.	批量操作成功后必须联动刷新。
5.	复制成功后必须联动刷新。
4.2 统一入口规范
1.	每个主实体定义一个 RelatedCaches 刷新入口。
2.	页面层不得散点调用多个缓存刷新函数。
3.	刷新函数要求幂等。
4.3 示例
1.	事件：CodeType 新增成功。
2.	动作：调用 RefreshRelatedCaches。
3.	结果：CodeType 相关视图缓存全部失效。
4.	影响：应用类型页新增代码类型可即时可见。
【5. 开发流程】
5.1 新需求落地步骤
1.	先做可变化判定。
2.	再确定放置层。
3.	优先扩展覆盖，不改生成层。
4.	补充回归用例。
5.	提交 PR 并勾选检查项。
5.2 重生成前后流程
1.	重生成前冻结业务改动。
2.	重生成后先编译，再跑最小回归。
3.	若行为差异，优先检查 Ex 覆盖是否失效。
【6. PR 检查清单】
1.	是否修改了生成层文件。
2.	若修改了生成层，是否给出 Ex 迁移方案。
3.	是否覆盖 Add/Update/Delete/Batch/Copy 流程。
4.	是否验证缓存联动。
5.	是否验证页面重进一致性。
6.	是否补充最小注释说明覆盖原因。
7.	是否通过编译与手测。
【7. 提交说明模板】
7.1 背景
1.	问题现象。
2.	影响范围。
7.2 方案
1.	变更放置层。
2.	关键设计点。
7.3 变更点
1.	覆盖方法列表。
2.	缓存联动入口。
7.4 验证
1.	用例一：新增。
2.	用例二：修改。
3.	用例三：删除。
4.	用例四：批量。
5.	用例五：复制。
7.5 风险与回滚
1.	风险点。
2.	回滚步骤。
【8. 代码评审准入规则】
1.	业务逻辑写入生成层，拒绝合并。
2.	缓存刷新散点分布，要求收敛。
3.	默认值多处定义，要求单点化。
4.	仅修单条流程未覆盖批量流程，拒绝合并。
【9. 最小回归套件】
1.	新增记录后跨页面可见。
2.	修改记录后列表即时生效。
3.	删除后关联页面不再出现。
4.	批量设置后联动缓存更新。
5.	复制后唯一性规则与联动缓存正确。
【10. 维护机制】
1.	每月回顾一次规范执行情况。
2.	每次生成器升级后更新文档版本。
3.	新增模块时复制本规范并做模块化差异补充。

【11. ExWApi 扩展钩子固定约定】
11.1 初始生成约定
1.	每个表的 ExWApi 文件必须初始生成 5 个后置钩子函数（空实现）。
2.	函数属于扩展层，默认可修改，初始状态为 no-op。
3.	禁止在生成层 WApi 中落地跨表缓存依赖。

11.2 函数清单（固定命名）
1.	AfterAddSuccess
2.	AfterUpdateSuccess
3.	AfterDeleteSuccess
4.	AfterBatchSuccess
5.	AfterCopySuccess

11.3 注释规范（函数级）
1.	层级：扩展层(Ex)
2.	生成状态：初始生成(空实现)
3.	可修改性：可修改
4.	职责：对应成功路径扩展点

11.4 样例模板
1.	示例：
export function XxxEx_AfterUpdateSuccess(): void {
	// 默认空实现(no-op)。
	// 如需启用联动刷新，请在此处调用 Xxx_ReFreshRelatedCaches();
}

【12. 各层函数清单（命名与职责）】
12.1 实体层（L0Entity，生成）
1.	SetXxx(value): 字段赋值并标记脏字段。
2.	GetFldValue(strFldName): 按字段名读取值。
3.	AttributeName: 字段名集合，用于通用映射与校验。

12.2 WApi 层（L3ForWApi，生成）
1.	Xxx_GetObjByKeyAsync: 按主键获取单条记录。
2.	Xxx_GetObjLstByPagerAsync: 分页查询。
3.	Xxx_AddNewRecordAsync: 新增记录。
4.	Xxx_UpdateRecordAsync: 修改记录。
5.	Xxx_DelRecordAsync / Xxx_DelXxxsAsync: 删除（单条/批量）。
6.	Xxx_CheckPropertyNew / Xxx_CheckProperty4Update: 新增/修改校验。
7.	(条件生成) Xxx_GetObjLstCache / Xxx_GetObjLstlocalStorage / Xxx_GetObjLstsessionStorage: 缓存读取函数（仅当该表启用缓存配置时生成）。
8.	(条件生成) Xxx_ReFreshThisCache / Xxx_GetLastRefreshTime: 缓存刷新与刷新时间函数（仅当该表启用缓存配置时生成）。
9.	(条件生成) Xxx_GetObjByKeyCache / Xxx_GetNameByKeyCache: 按主键缓存读取函数（仅当该表启用缓存配置时生成）。
说明：WApi 只承载通用数据访问；缓存函数是否生成由表配置决定；跨表业务联动不放在生成层 WApi 中。

12.3 WApiEx 层（L3ForWApiEx，初始生成+可修改）
1.	XxxEx_CopyToEx: 基类对象复制到扩展对象。
2.	XxxEx_FuncMapByFldName: 扩展字段函数映射入口。
3.	XxxEx_GetObjExLstByPagerAsync: 扩展对象分页。
4.	Xxx_ReFreshRelatedCaches: 相关缓存刷新入口（可选，推荐）。
5.	XxxEx_AfterAddSuccess / AfterUpdateSuccess / AfterDeleteSuccess / AfterBatchSuccess / AfterCopySuccess: 成功后扩展钩子（初始空实现）。

12.4 视图基类（viewsBase，生成）
1.	XxxCRUD: BindGv_*, BindTab_*, DelRecord, DelMultiRecord, Set* 批量设置。
2.	Xxx_Edit: AddNewRecordSave, AddNewRecordWithMaxIdSave, UpdateRecordSave, ShowDataFromObj, GetEditDataObj。
说明：基类流程可重生成，不放项目特有策略。

12.5 视图扩展层（views/*Ex.ts，可修改）
1.	XxxCRUDEx: 排序、自定义列表展示、批量策略、成功后联动刷新。
2.	Xxx_EditEx: 覆盖保存成功路径并调用 After* 或 ReFreshRelatedCaches。
3.	btn_Click / btnEdit_Click: 命令分发与页面行为路由。
说明：业务规则、跨表联动、交互差异统一放 Ex。

12.6 界面层（.vue，可修改）
1.	BindDdl4QryRegion / BindDdl4FeatureRegion: 下拉与默认值初始化。
2.	btnQuery_Click / btnCreate_Click / btnUpdate_Click / btnDelete_Click: 入口事件。
3.	onMounted: 页面启动编排（绑定、实例化、初次加载）。
说明：界面层负责编排与交互，不直接承载底层数据规则。
