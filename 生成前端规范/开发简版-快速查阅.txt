[AGC 前端生成代码协作规则 - 开发简版]

1. 目标
- 代码可重生成。
- 业务行为不丢失。
- 缓存联动一致。

2. 放置规则
- 生成层: 只放标准模板能力。
- Ex 扩展层: 只放业务变化能力。
- 禁止把业务逻辑写入生成层文件。

3. 可变化项 (放 Ex)
- 默认值。
- 筛选与排序策略。
- 弹窗和同页编辑流程。
- 跨表缓存联动。

4. 不可变化项 (放生成层)
- 字段契约。
- 接口参数和返回结构。
- 主键与唯一性基础约束。

5. 缓存联动必做
- Add 成功后刷新。
- Update 成功后刷新。
- Delete 成功后刷新。
- Batch 成功后刷新。
- Copy 成功后刷新。

6. PR 最小检查
- 是否修改了生成层。
- 是否可重生成不丢行为。
- 是否覆盖 Add/Update/Delete/Batch/Copy。
- 是否验证联动页面已更新。

7. 一句话原则
- 会变的放 Ex，不会变的留生成层。

8. ExWApi 固定生成约定
- 每个表初始生成 5 个 After*Success 空函数。
- 函数默认 no-op，后续按业务启用。
- 命名固定: AfterAddSuccess / AfterUpdateSuccess / AfterDeleteSuccess / AfterBatchSuccess / AfterCopySuccess。
